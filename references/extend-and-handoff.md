# 扩展与协作（超出本技能范围时：衔接，不拒绝）

frontend-gen 专注"生成/完善前端项目"，但用户可能从任何方向切入——要部署上线、要接/生成后端、要按 Figma 还原 UI。遇到这些**不要停在"我不做"**，而是接住它，让用户在本技能里把项目从一句话推到上线。

## 通用协作协议（三步）

1. **先看手里有什么**：这项能力是否已作为 skill 或 MCP 可用（看可用技能列表 / 已配置的 MCP）？有就直接用。
2. **没有就"发现 + 征得同意后安装"**：去 skill 清单（如 awesome-claude-skills）或 MCP 注册表找到**核实过的** `owner/repo` 或 server，**先告诉用户要装什么、为什么**，同意后再装——skill 用 `npx skills add <owner/repo> -a claude-code`（项目级）或 `-g`（全局）；MCP 引导在 Claude 配置里添加该 server。**不要硬编码没核实过的仓库名。**
3. **衔接执行**：用该 skill/MCP 完成那一段，再回到本技能继续完善项目，并在 `docs/frontend-map.md` 记一句（用了什么部署/后端/设计源）。

## 场景 A：部署 / 上线

- 触发：部署、上线、deploy、发布到网上、放服务器。
- 产物是可构建项目，部署是交接。按栈选最省事路径：
  - 纯静态 / SSG（官网、作品集）→ **Netlify 拖拽**、**Cloudflare Pages / GitHub Pages**（小白最省事）。
  - Next.js（含 SSR/API）→ **Vercel**（连 GitHub 一键）。
- 做法：有部署类 skill/CLI 就用；否则给小白**能照做的图文步骤**（注册 → 连仓库或拖 build 产物 → 拿到网址），或征得同意装一个部署 skill。
- 诚实：部署需登录账号/授权，技能不能替造凭据；引导用户登录，密钥由用户自己填，别写进代码库。

## 场景 B：后端 / 真实接口 / 生成后端代码

- 触发：接后端、要数据库、登录、支付、真实接口、把假数据换成真的、生成后端代码。
- 本技能是纯前端 + 模拟数据，但**刻意把数据藏在 `lib/content.ts` 的类型契约后面**——这就是对接后端的"接缝"：后端到位时只换数据来源，组件和页面不动。
- 做法：
  - 对接**现有 API**：把 `lib/content.ts` 里的 mock 换成 fetch 真实接口，保持返回类型一致，页面零改动；base URL/key 放 env。
  - **轻量后端**（表单提交、简单读写）：可用 Next API routes / server actions 起个壳；但真正的业务逻辑/数据库/鉴权交给后端类 skill。
  - **独立后端服务 / 生成后端代码**：发现并（征得同意后）安装或调用后端生成类 skill 来做；本技能保持数据契约稳定，让前后端对得上。
- 始终讲清前端怎么接：接口形状、env 配置、加载态与错误态。

## 场景 C：按 Figma 还原 UI

- 触发：按这个 Figma 做、还原设计稿、figma.com 链接、设计稿转代码、拉图层/图片/图标。
- 做法：
  1. **接入 Figma**：引导用户添加官方 **Figma MCP（Dev Mode MCP）** server 并在 Figma 打开目标文件 / 选中节点；或使用环境里已有的 figma 相关 skill（如 `figma-use`、`figma-generate-design`）。
  2. **取设计上下文**：通过 MCP/skill 拉节点结构、尺寸、颜色与字体变量、间距，并导出图片与 SVG 图标。
  3. **映射进设计令牌系统**：把 Figma 的 variables/styles **对应成 `:root` 令牌**（颜色/字号/间距/圆角），图片图标导出到 `public/`——还原出的 UI 天然进入本技能的令牌体系，跨页一致、可续建（见 `references/design-system.md`）。
  4. 用本技能的组件/数据/路由把稿子落成**可维护项目**，而不是把一张稿硬塞进一个组件。
- 没接 MCP 时：让用户给截图/导出图 + 关键数值，按 `references/source-intake.md` 提取，照样能做，只是不如 MCP 精确。

## 原则

- **任何方向切入都接得住**：一句话、一张 Figma、一个旧站、一份 API 文档——都先归到"生成或完善这个前端项目"，缺的能力按上面协议补齐，而不是让用户卡在边界。
- 装任何 skill/MCP 前先说清装什么、为什么，**征得同意**；账号/密钥由用户提供。
- 交接出去的部分回写 `docs/frontend-map.md`，下次接力有据可查。
